	## 3. 典型失败案例分析
	### 案例 1: 图片模糊不清
	*   **文件名**: `blur_cert_01.jpg`
	*   **错误原因**: 网络请求超时 (Timeout > 60s)
	*   **建议**: 前端限制上传图片的最小分辨率，或增加图片预处理（锐化）步骤。
	### 案例 2: 扫描件倾斜严重
	*   **文件名**: `rotated_scan.pdf`
	*   **错误原因**: 返回内容非JSON格式 (包含大量无关文本)
	*   **建议**: 在发送API前，强制调用自动旋转校正接口。
	### 案例 3: 字段缺失
	*   **文件名**: `simple_cert.png`
	*   **现象**: 成功返回JSON，但 `organizer` 和 `tutor_name` 为空。
	*   **原因**: 原证书上确实未印制指导教师姓名。
	*   **处理**: 代码已自动将空值置为 `""`，并标记为 `warning`，不影响提交流程。
	---
	## 4. 总结与建议
	1.  **整体表现**: GLM-4V 在结构化证书识别方面表现优异，核心字段（姓名、奖项）识别准确率极高。
	2.  **优化建议**:
	    *   对于模糊图片，建议在前端增加“图片质量检测”环节。
	    *   针对 `competition_type`（A/B类）这种证书上往往不直接写明的信息，建议不依赖OCR识别，改为让用户手动选择下拉框。
	    *   增加重试机制：当遇到网络抖动导致的 `500` 错误时，自动重试1次。
